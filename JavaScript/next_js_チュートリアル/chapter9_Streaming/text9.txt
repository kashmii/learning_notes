# ストリーミングとは
ルートを小さな "チャンク "に分割し、準備ができ次第、サーバーからクライアントに順次ストリーミングするデータ転送技術である。

ウォーターフォール・アプローチ：　すべてのデータの準備が整うまでUIのレンダリングがブロックされる
ストリーミング・アプローチでは、データが利用可能になるとすぐにUIをレンダリングすることができる

Next.jsでストリーミングを行う方法は2種類
1. ページレベルでは、loading.tsxファイル
2. 特定のコンポーネントには、<Suspense>を使用します

...


### loading.tsx
・ページのローディング状態を管理するためのファイル（Next.jsのルールとして、このファイル名にすればそのようになる）

### A loading skeleton
... a simplified version of the UI.

## Route groups
ルートグループは、特定のルートのサブセットをグループ化するために使用される。

アプリディレクトリでは、ネストされたフォルダは通常 URL パスにマップされます。
しかし、フォルダをルートグループとしてマークすることで、そのフォルダがルートのURLパスに含まれないようにすることができます

### React Suspense

SuspenseはReactの機能で、コンポーネントが非同期にロードされる間に一時的な"fallback"コンテンツを表示することができます。

ページをブロックする代わりに、Susppenseを使ってこのコンポーネントだけをストリーミングし、
ページの残りのUIをすぐに表示することができます。

・suspenseでラッパーされているところは、非同期にロードされる
<Suspense fallback={<LatestInvoicesSkeleton />}>
  <LatestInvoices />
</Suspense>
